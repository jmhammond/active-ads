There is a one to one correspondence between ordered rooted trees and binary trees. If you start with an ordered rooted tree, \(T\text{,}\) you can build a binary tree \(B\) with an empty right subtree by placing the the root of \(T\) at the root of \(B\text{.}\) Then for every vertex \(v\) from \(T\) that has been placed in \(B\text{,}\) place it’s leftmost child (if there is one) as \(v\)’s left child in \(B\text{.}\) Make \(v\)’s next sibling (if there is one) in \(T\) the right child in \(B\text{.}\)
